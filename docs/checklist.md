# 黑马点评实现检查清单

这份清单用来防止“功能好像写了，但少验证一半”。每完成一个阶段，就按这里勾一遍。

## 阶段 0：环境

- [ ] MySQL `hmdp` 数据库已创建。
- [ ] `hmdp.sql` 已导入。
- [ ] 后端能启动在 `8081`。
- [ ] `/shop-type/list` 能返回数据。
- [ ] nginx 能启动在 `8080`。
- [ ] 前端 `/api/shop-type/list` 能代理到后端。

## 阶段 1：基础接口

- [ ] 能讲清 Controller-Service-Mapper。
- [ ] `ShopTypeController` 查询已能走 Service。
- [ ] `ShopController#queryShopById` 有参数校验。
- [ ] 不存在的店铺 id 返回失败。
- [ ] `mvn -q -DskipTests compile` 通过。

## 阶段 2：登录

- [ ] 发送验证码能校验手机号。
- [ ] 验证码能保存到 Redis。
- [ ] 登录能校验验证码。
- [ ] 新手机号能自动注册。
- [ ] 登录成功返回 token。
- [ ] Redis 有 `login:token:{token}`。
- [ ] `/user/me` 能返回当前用户。
- [ ] 未登录访问受保护接口返回 401。
- [ ] 请求结束会清理 `UserHolder`。

## 阶段 3：缓存

- [ ] 商铺详情先查 Redis。
- [ ] 缓存未命中会查 MySQL 并写入 Redis。
- [ ] 不存在店铺会缓存空值。
- [ ] 更新商铺后删除缓存。
- [ ] 能解释缓存穿透。
- [ ] 能解释缓存击穿。
- [ ] 能解释逻辑过期适合什么场景。

## 阶段 4：社交

- [ ] 热门博客能分页查询。
- [ ] 博客能补作者昵称和头像。
- [ ] 点赞能增加数据库点赞数。
- [ ] Redis ZSet 能记录点赞用户。
- [ ] 再次点赞能取消。
- [ ] 能查询前 5 个点赞用户。
- [ ] 关注/取关能同步数据库和 Redis Set。
- [ ] 共同关注能用 Set 交集实现。
- [ ] 发布博客能推送到粉丝 Feed。
- [ ] 关注流滚动分页不重复。

## 阶段 5：高级 Redis

- [ ] 签到能写入 Bitmap。
- [ ] 连续签到统计正确。
- [ ] 商铺 GEO 数据已导入 Redis。
- [ ] 附近商铺能按距离返回。
- [ ] 距离字段能返回给前端。

## 阶段 6：秒杀基础

- [ ] 能判断秒杀开始/结束。
- [ ] 能判断库存。
- [ ] 扣库存使用 `stock > 0` 条件。
- [ ] 能防止同一用户重复下单。
- [ ] 下单方法有事务。
- [ ] 数据库有唯一索引兜底。
- [ ] 并发测试后库存不小于 0。

## 阶段 7：高级秒杀

- [ ] Redis 库存已初始化。
- [ ] Stream 消费组已创建。
- [ ] Lua 能原子判断库存和一人一单。
- [ ] 接口线程只做资格判断并返回订单 id。
- [ ] 后台线程能消费 Stream。
- [ ] 成功处理后 ACK。
- [ ] 异常能处理 Pending List。
- [ ] 数据库仍有唯一索引兜底。

## 阶段 8：工程化

- [ ] 有接口测试记录。
- [ ] 有常见问题排查文档。
- [ ] 有 README 或学习说明。
- [ ] 能用一句话介绍项目。
- [ ] 能讲 3 个 Redis 数据结构选择。
- [ ] 能讲缓存穿透、击穿、雪崩。
- [ ] 能讲秒杀从基础版到高级版的演进。
