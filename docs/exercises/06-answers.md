# 阶段 6 参考答案：秒杀基础版

## 概念题参考答案

1. 超卖是实际卖出的数量超过库存。高并发下多个线程可能同时读到库存充足，然后都扣减并创建订单。
2. 核心是更新库存时带上 `stock > 0` 条件，例如 `update tb_seckill_voucher set stock = stock - 1 where voucher_id = ? and stock > 0`。
3. 两个并发请求可能同时查到没有订单，然后都创建订单。需要加锁或数据库唯一索引兜底。
4. 写在 private 方法、同类自调用、异常被吞、方法不是 Spring Bean 调用等情况都可能导致事务不生效。
5. JVM 锁只在单个进程内有效，多台服务实例各有自己的锁，不能互相感知。

## 代码阅读题参考答案

1. `SeckillVoucher` 主要多了 `stock`、`beginTime`、`endTime` 等秒杀库存和时间窗口字段。
2. 订单至少需要订单 id、用户 id、券 id、创建时间等字段。

## 小改造参考

核心顺序：

1. 查秒杀券。
2. 判断开始和结束时间。
3. 判断库存。
4. 判断当前用户是否买过。
5. `stock = stock - 1` 且 `stock > 0`。
6. 创建订单。
7. 返回订单 id。

## 面试表达参考

“我没有使用先查库存再直接扣减的方式，因为并发下多个线程可能都看到库存大于 0。我的做法是在更新库存时加入 `stock > 0` 条件，让数据库在扣减瞬间再次判断库存是否充足。只有更新成功才创建订单，如果更新影响行数为 0，就说明库存不足。这样可以避免库存被扣成负数。”
