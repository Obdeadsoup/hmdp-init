# 学完后能得到什么版本

这份文档回答三个问题：

1. 按当前教程做完后，项目会达到什么程度？
2. Redis 和后端开发核心内容有没有覆盖？
3. 哪些内容属于“基础功能版”，哪些属于后续优化？

## 结论

按 `docs/learning/01` 到 `docs/learning/08` 逐步实现、验证、完成练习后，你可以得到一个“学习版完整黑马点评后端项目”。

它会具备黑马点评实战项目的主要后端功能：

- 商铺类型和商铺查询。
- 手机号验证码登录。
- Redis Token 登录态。
- 登录拦截器和 ThreadLocal 用户上下文。
- 商铺详情缓存。
- 缓存穿透、击穿、逻辑过期。
- 博客热门列表、详情、点赞。
- 关注、取关、共同关注。
- Feed 流。
- Bitmap 签到。
- GEO 附近商铺。
- 优惠券秒杀基础版。
- Lua + Redis Stream 异步秒杀进阶版。
- 接口调试、排错、部署和面试表达。

换句话说：它不是只教“基础 CRUD”，而是覆盖了一个后端学习项目最有价值的 Redis 和高并发部分。

## 它是什么级别

### 可以作为

- Java 后端入门到进阶学习项目。
- Redis 实战学习项目。
- Spring Boot + MyBatis-Plus 综合练习。
- 简历项目基础版本。
- 面试表达训练材料。
- Agent 协作开发练习仓库。

### 还不是

- 真正生产级点评系统。
- 完整商业电商秒杀系统。
- 高可用分布式系统。
- 严格安全合规系统。
- 完整 DevOps 平台。

这不是坏事。学习路线应该先形成主干，再做增强。你现在最重要的是把主链路写出来、跑起来、讲清楚。

## 做完后的验收标准

你不需要追求“一口气全部写完”。每个阶段完成后，用下面 5 个问题验收：

1. 功能是否能通过接口验证？
2. 数据库或 Redis 里的数据是否符合预期？
3. 我能不能画出调用链？
4. 我能不能说出这个设计解决了什么问题？
5. 我能不能回答对应练习题？

## 基础版和增强版边界

### 基础版必须完成

- 阶段 1：基础接口和分层。
- 阶段 2：Redis Token 登录。
- 阶段 3：商铺缓存和缓存穿透。
- 阶段 4：点赞、关注、共同关注。
- 阶段 5：签到和 GEO。
- 阶段 6：秒杀基础版。

### 进阶版建议完成

- 阶段 3：互斥锁、逻辑过期、CacheClient。
- 阶段 4：Feed 流滚动分页。
- 阶段 7：Lua + Redis Stream 异步秒杀。
- 阶段 8：测试、部署、面试表达。

### 后续优化再完成

- 接口权限系统。
- 操作日志。
- 限流和防刷。
- OpenAPI 文档。
- Docker Compose 完整环境。
- GitHub Actions / CI。
- Redis Cluster。
- 消息可靠性补偿表。
- 管理后台。
- 搜索服务。

这些内容放在 `10-advanced-optimization-roadmap.md`。
