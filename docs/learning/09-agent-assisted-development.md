# 阶段 9：通过黑马点评学习 Agent 协作开发

## 本阶段目标

你已经把黑马点评当作后端项目来学。接下来可以把它进一步升级成“Agent 协作开发训练场”：让 ChatGPT / Codex 帮你读代码、拆任务、审查实现、生成测试、定位报错，但最终你仍然掌握设计和代码。

本阶段不是让 AI 直接替你写完整项目，而是学习如何把 AI 变成靠谱的结对工程师。

## 你要学会什么

1. 如何给 Agent 写清楚项目背景和约束。
2. 如何把一个大需求拆成可验证的小任务。
3. 如何让 Agent 先读代码再建议修改。
4. 如何让 Agent 生成代码审查意见。
5. 如何让 Agent 按排错流程定位问题。
6. 如何让 Agent 维护学习文档和接口测试记录。
7. 如何判断 Agent 的回答是否可靠。

## Agent 协作的基本原则

### 1. 先给边界，再给任务

不要只说：

```text
帮我实现登录。
```

更好的说法：

```text
当前是黑马点评 init 分支，我正在学习阶段 2：登录模块。请先读取 AGENT.md 和 UserController、UserServiceImpl、RedisConstants、UserHolder。不要直接大改代码，先告诉我文件职责、Session 登录和 Redis Token 登录区别，再给最小实现计划。
```

### 2. 让 Agent 解释再实现

你要避免“代码跑了但自己不懂”。每次都要求：

- 文件地图。
- 核心概念。
- 修改计划。
- 代码修改。
- 验证方式。
- 常见坑。
- 小结。

### 3. 每次只让 Agent 做一个小闭环

错误示例：

```text
帮我实现登录、缓存、点赞、秒杀。
```

正确示例：

```text
只实现发送验证码：手机号校验、生成 6 位验证码、写入 Redis、控制台打印验证码、接口验证。不要实现登录。
```

### 4. 让 Agent 给出可验证证据

每次代码完成后，让它告诉你：

- 改了哪些文件。
- 用什么命令编译。
- 用什么接口验证。
- Redis / MySQL 怎么检查。
- 哪些情况还没验证。

## 推荐 Agent 工作流

### 工作流 A：学习一个新模块

1. 让 Agent 读相关文件。
2. 让 Agent 画调用链。
3. 让 Agent 用初学者语言解释概念。
4. 让 Agent 拆成 3 到 5 个小任务。
5. 你按任务逐步实现。
6. 每实现一步，让 Agent 审查。
7. 最后让 Agent 出题考你。

适合：

- 登录。
- 缓存。
- 秒杀。
- Feed 流。

### 工作流 B：实现一个小功能

1. 描述当前阶段和目标。
2. 限定只改哪些文件。
3. 要求先给计划。
4. 允许 Agent 修改。
5. 编译和接口验证。
6. 让 Agent 总结知识点。

适合：

- `/user/me`
- 商铺详情缓存
- 点赞接口
- 签到接口

### 工作流 C：排错

1. 粘贴关键报错，不要贴一万行。
2. 告诉 Agent 你执行过什么命令。
3. 告诉 Agent 最近改过哪些文件。
4. 要求它按错误类型分类。
5. 先修最可能的一处。

适合：

- 启动失败。
- Redis 连接失败。
- SQL 异常。
- 401 未登录。
- 秒杀重复下单。

### 工作流 D：代码审查

让 Agent 按这些角度检查：

- 功能是否正确。
- 边界条件是否完整。
- 是否符合分层。
- 是否有并发风险。
- 是否有事务风险。
- 是否有缓存一致性风险。
- 是否泄露敏感信息。
- 是否有必要补测试。

## 在本项目中适合交给 Agent 的任务

### 适合

- 读代码并画调用链。
- 解释 Spring / Redis / MyBatis-Plus 概念。
- 生成小范围实现计划。
- 补文档。
- 补接口测试模板。
- 代码审查。
- 排错定位。
- 生成面试问答。
- 生成测试用例草稿。

### 不适合直接交给 Agent

- 一次性生成整个项目。
- 不读现有代码就重构架构。
- 直接复制完整参考实现。
- 直接改数据库结构而不说明影响。
- 在你没理解前引入复杂依赖。
- 跳过验证。

## Agent 开发任务设计模板

每个任务尽量写成这样：

```text
背景：
我在黑马点评项目，当前阶段是 X，目标是 Y。

已有文件：
- ...
- ...

任务边界：
本次只做 A，不做 B/C。

要求：
1. 先读取相关文件并给文件地图。
2. 用初学者能懂的话解释核心概念。
3. 给最小修改计划。
4. 修改后运行编译。
5. 给接口验证方式。
6. 总结我必须掌握的点。
```

## Agent 产出质量检查

Agent 给你的方案，至少要满足：

1. 说得出改哪些文件。
2. 说得出为什么改。
3. 不违反项目分层。
4. 不把真实密码写进代码。
5. 能编译。
6. 有接口验证方式。
7. 有失败场景说明。
8. 你能复述核心逻辑。

如果第 8 条做不到，说明还没学会，不要急着下一阶段。

## 学完本阶段后你应该能做到

- 把一个后端功能拆成 Agent 可执行的小任务。
- 让 Agent 先读代码再行动。
- 用 Agent 做排错和审查，而不是盲目接受答案。
- 维护项目级教学文档。
- 把 AI 协作能力写进自己的项目经验里。
