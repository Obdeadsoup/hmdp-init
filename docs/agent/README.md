# Agent 协作开发说明

这个目录用于把黑马点评项目变成一个 AI Agent 协作开发练习场。你可以用 ChatGPT / Codex 辅助学习、实现、排错、审查和文档维护。

## 推荐使用方式

1. 先读根目录 `AGENT.md`，了解本项目的协作原则。
2. 按 `docs/learning` 的阶段推进。
3. 每次问 Agent 前，先从 `prompt-templates.md` 复制一个合适模板。
4. 让 Agent 输出文件地图、概念解释、最小计划、验证方式。
5. 不理解的代码不要直接保留，继续追问直到能复述。

## Agent 角色分工

### 后端导师

用于解释：

- Spring Boot。
- MyBatis-Plus。
- Redis。
- 事务。
- 分布式锁。
- Lua / Stream。

### 结对工程师

用于协作实现：

- 小功能。
- 接口补全。
- 缓存改造。
- 秒杀链路。
- 测试和文档。

### 代码审查员

用于检查：

- 分层是否合理。
- 边界条件是否完整。
- 并发和事务风险。
- 缓存一致性问题。
- 是否泄露敏感信息。

### 排错助手

用于定位：

- 编译错误。
- 启动错误。
- 数据库错误。
- Redis 错误。
- 接口错误。
- 逻辑错误。

## Agent 任务粒度

### 好任务

```text
只实现 Redis 验证码发送，不实现登录。
```

```text
审查 ShopServiceImpl 的缓存穿透处理，重点看空值缓存 TTL 和更新后删缓存。
```

```text
根据这段报错判断 Redis Stream 消费组为什么不存在，并给最小修复命令。
```

### 差任务

```text
帮我写完整黑马点评。
```

```text
直接把所有 Redis 功能都加上。
```

```text
不用解释，能跑就行。
```

## 每次让 Agent 改代码前的检查

- [ ] 当前阶段说清楚了吗？
- [ ] 本次只做一个明确目标吗？
- [ ] 相关文件列出来了吗？
- [ ] 明确不做什么了吗？
- [ ] 要求验证了吗？
- [ ] 要求总结知识点了吗？

## 每次 Agent 改完后的检查

- [ ] 编译通过了吗？
- [ ] 接口验证了吗？
- [ ] 数据库或 Redis 检查了吗？
- [ ] 文件改动是否过大？
- [ ] 是否引入不必要依赖？
- [ ] 是否有真实密码或 IP？
- [ ] 你能解释主要代码吗？

## 可写进简历的表达

可以这样表达 AI 协作能力，但不要夸大：

```text
在项目学习过程中，我将需求拆分为阶段化任务，并通过 AI Agent 辅助进行代码阅读、实现计划、错误排查、代码审查和文档维护。每次协作都要求先定位文件、解释设计、给出验证方式，避免直接复制代码，提升了独立排错和项目表达能力。
```
